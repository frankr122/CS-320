# CS-320

How can I ensure that my code, program, or software is functional and secure?
I ensure my code is functional and secure by carefully following requirements, validating all user inputs, and writing thorough unit tests that check both normal behavior and edge cases before considering the work complete.

How do I interpret user needs and incorporate them into a program?
I interpret user needs by breaking down the requirements into smaller, testable pieces of functionality and then building methods and validations that directly reflect what the user expects the program to do.

How do I approach designing software?
I approach designing software by keeping it organized and modular, separating responsibilities into clear classes and services so the code is easier to test, maintain, and improve over time.
